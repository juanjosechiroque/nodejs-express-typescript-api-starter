# Security Policy

## Reporting a vulnerability

Do not disclose suspected vulnerabilities in a public issue. Use [GitHub private vulnerability reporting](https://github.com/juanjosechiroque/nodejs-express-typescript-api-starter/security/advisories/new) so the report and remediation can be coordinated privately.

Include the affected version, reproduction steps, impact, and any suggested mitigation. Never include real credentials, access tokens, customer data, or other production secrets in a report.

## Supported versions

Security fixes are applied to the latest release on the `main` branch. Projects created from this starter own their dependency updates, secret rotation, infrastructure hardening, and vulnerability-response process after adoption.
